## 一、软件简介与系统要求

**Steam Tool Plus** 是一款纯本地运行的 Steam 游戏管理工具，完全绿色便携，无需安装，所有数据保存在程序运行目录与 `%appdata%/SteamToolPlus` 配置目录下。

### 主要功能

- **游戏浏览**：以 Steam 风格网格浏览游戏，支持搜索与分页
- **游戏本体下载**：通过 manifest 清单文件直连 Steam 官方服务器下载正版分流文件，支持断点续传与完整性验证
- **免 Steam 补丁注入**：为游戏注入模拟 Steam 运行环境，无需 Steam 客户端即可启动游戏
- **清单入库**：将清单文件导入 Steam，使 Steam 库中显示对应游戏
- **游戏库管理**：统一管理本地游戏，支持添加、删除、启动、统计游玩时长
- **局域网联机**：配置局域网多人联机参数
- **完整配置管理器**：统一配置 DLC、成就、统计、物品、模组、控制器、用户、主配置、Overlay、ColdClient、Lobby、应用分组、音视频、Depots/Branches、DLC 网络、HTTP 物品等所有补丁参数
- **假入库工具**：在 Steam 库中模拟显示已拥有游戏，无需真实清单文件
- **主题与背景**：支持深色/浅色主题与自定义背景图片
- **多语言支持**：内置简体中文与 English

### 系统要求

- **操作系统**：Windows 10/11 64 位
- **运行环境**：无需 Steam 客户端，无需联网（除下载游戏与封面外）
- **硬件要求**：最低 4GB 内存即可流畅运行
- **存储空间**：程序本体 ≤ 10MB，游戏数据另行计算

## 二、首次使用

1. **下载程序**：从上方网盘下载最新版本 `SteamToolPlus.exe`
2. **解压文件**：将程序解压到任意目录，**建议使用纯英文路径**，避免部分游戏因中文路径导致启动异常
3. **运行程序**：双击 `SteamToolPlus.exe`
4. **首次配置弹窗**：程序首次启动时会显示配置引导，可配置 Steam 路径、默认下载路径等；也可点击跳过，稍后手动进入【设置】→【全局设置】配置
5. **配置文件目录**：程序会自动在 `%appdata%/SteamToolPlus` 目录创建配置文件，所有用户数据均保存在此处

### 首次配置弹窗说明

- **Steam 路径**：设置 Steam 安装目录，用于清单入库功能
- **下载路径**：设置游戏默认下载路径，程序会优先使用 D 盘并自动创建 `SteamGame/游戏名` 目录
- **背景图片**：选择个性化背景主题
- **跳过配置**：可关闭弹窗，后续手动配置

### 二维码弹窗

在游戏详情页点击网盘下载按钮时，会弹出二维码，方便手机扫码访问网盘链接。点击弹窗外区域或关闭按钮即可关闭。

## 四、浏览与下载游戏

### 4.1 进入游戏详情页

1. 点击顶部菜单 **游戏** → **浏览**
2. 在搜索框输入游戏名称或 ID 查找游戏
3. 点击游戏卡片进入 **游戏详情页**

### 4.2 游戏详情页布局

游戏详情页分为三个区域：

- **顶部**：返回按钮、游戏标题、游戏 ID、补丁类型标签
- **中部左侧**：游戏封面大图（图片来自 `resources/pic/Game_Cover/`，尺寸 460×215）
- **中部右侧**：游戏路径选择、下载路径选择
- **底部**：标签页切换（下载、补丁等）

### 4.3 下载游戏本体

#### 前置准备

1. 从网盘下载游戏的清单文件压缩包（通常为 7z 格式）
2. 将清单文件解压到本地，确保文件夹内包含 `.vdf` / `.lua` 和 `.manifest` 文件

#### 方法一：通过游戏详情页下载

1. 进入游戏详情页，切换到 **下载** 标签
2. 在“清单文件选择”区域选择来源类型：
   - **7z 文件**：直接选择清单压缩包
   - **文件夹**：选择包含 `.vdf` / `.lua` 和 `.manifest` 的文件夹
3. 点击“选择 7z 文件”或“选择文件夹”
4. 在“游戏下载路径”处点击 **浏览**，选择游戏下载目录
5. 点击 **开始下载**
6. 查看下载进度、日志输出

> **说明**：方法一【开始下载】通过清单文件直连 Steam 官方服务器下载正版分流文件，下载完成后需要 **注入补丁** 才能游玩。

#### 方法二：通过工具 → 本体下载

1. 点击顶部菜单 **工具** → **本体下载**
2. 切换到“单游戏下载”模式
3. 选择清单文件夹或压缩包
4. 程序自动提取游戏信息并设置下载路径
5. 点击 **开始下载**

#### 批量下载

1. 在“本体下载”页面切换到“批量下载”模式
2. 选择包含多个游戏清单文件夹的父文件夹
3. 程序自动扫描并列出所有可下载的游戏
4. 勾选需要下载的游戏
5. 设置统一下载路径
6. 点击 **批量下载**
7. 可选择下载完成后自动关机

### 4.4 下载控制与日志

- **日志目录**：`%appdata%/SteamToolPlus/log/{game_id}/`
- **断点续传**：若下载进程异常退出，程序会自动检测未完成的 depot 并重新启动续传
- **暂停下载**：点击 **暂停** 按钮可停止当前下载进程
- **完整性验证**：下载完成后，可在游戏详情页点击 **验证游戏完整性** 按钮重新校验文件
- **下载完成**：程序会自动清理已完成的 JSON 进度文件

## 五、游戏库管理

### 5.1 进入游戏库

1. 点击顶部菜单 **游戏** → **库**
2. 左侧显示已下载或导入的游戏列表
3. 右侧显示选中游戏的详细信息与大图背景

### 5.2 添加游戏到库

- **自动添加**：通过“本体下载”完成的游戏会自动添加到库
- **手动导入**：点击 **导入游戏** 按钮，选择游戏安装目录和主程序 exe，填写游戏信息（名称、Steam ID 等），点击导入

### 5.3 启动与关闭游戏

1. 在游戏卡片或详情面板点击 **开始游戏**
2. 程序自动记录启动时间并实时统计游玩时长
3. 游戏运行时按钮变为 **关闭游戏**，点击可结束游戏进程

### 5.4 管理游戏

右键游戏卡片可调出操作菜单：

- **编辑游戏**：修改游戏目录、exe 路径、存档路径、封面
- **浏览本地文件**：打开游戏所在文件夹
- **收藏/取消收藏**：收藏的游戏显示在列表顶部
- **卸载游戏**：删除游戏文件，但保留游戏数据记录

## 六、免 Steam 补丁注入

免 Steam 补丁功能可为游戏注入模拟 Steam 运行环境，无需 Steam 客户端即可启动游戏。

### 6.1 进入补丁页面

点击顶部菜单 **工具** → **免Steam补丁**。

> **状态记忆**：补丁页面会记住您填写的游戏路径、AppID、模拟器模式、DLL 版本、是否使用实验版等状态。只有关闭程序或点击 **重置** 按钮，状态才会恢复初始。

### 6.2 基础配置流程

#### 步骤一：选择游戏文件夹

1. 点击 **选择游戏文件夹** 按钮
2. 浏览并选择游戏安装目录（该目录应包含 `steam_api.dll` 或 `steam_api64.dll`）
3. 程序自动检测文件夹中的 DLL 文件

#### 步骤二：选择模拟器模式

| 模式 | 说明 |
|------|------|
| **标准模式** | 替换 `steam_api.dll`，适用于大多数 Steam 游戏 |
| **高级模式** | 替换 `steamclient.dll` + `steam_api.dll`，适用于特殊保护游戏 |

#### 步骤三：选择 DLL 版本（标准模式下）

| 版本 | 说明 |
|------|------|
| **稳定版** | 推荐大多数用户使用，兼容性好，稳定性高 |
| **功能版（实验版）** | 包含 Overlay 等高级功能，部分游戏可能有兼容性问题 |

> **注意**：只有在标准模式下勾选了“使用实验版（功能版）”，完整配置管理器中的 **Overlay 配置** 入口才会启用。

#### 步骤四：配置 Steam App ID

1. 在 **Steam App ID** 输入框中填写游戏的 App ID
2. 可点击 **SteamDB** 链接在浏览器查询游戏 App ID
3. 程序会自动生成 `steam_interfaces.txt` 文件

#### 步骤五：应用基础配置

1. 点击 **应用配置** 按钮
2. 程序自动在游戏目录创建 `steam_settings` 文件夹并写入默认配置：
   - `configs.main.ini`：主配置
   - `configs.user.ini`：用户配置
   - `configs.app.ini`：DLC 与 Depot 配置（默认解锁全部 DLC）
   - `configs.overlay.ini`：Overlay 配置
3. 自动复制补丁 DLL 到游戏目录
4. 自动备份原文件（添加 `.bak` 后缀）

> **默认值说明**：
> - **存档路径**默认使用 `%appdata%/GSE Saves`
> - **DLC** 默认解锁全部 DLC，不会变为手动指定
> - 所有可选项均不会自动填充示例值

### 6.3 完整配置管理器

点击 **完整配置管理器** 按钮，可统一配置所有补丁参数。管理器左侧为导航列表，右侧为配置表单。

2. 点击 **Steamless 脱壳** 按钮
3. 程序自动备份原 exe 文件并生成脱壳后的 exe

## 七、清单入库

将清单文件导入 Steam，使 Steam 库中显示对应游戏。本程序使用 **OpenSteamTool** 内核完成入库，支持热加载、访问令牌、成就数据 SteamID 等高级功能。

### 7.1 前置要求

- 在【全局设置】中配置正确的 **Steam 安装路径**
- 首次使用会自动将 OpenSteamTool 内核 DLL 安装到 Steam 根目录

### 7.2 使用步骤

1. 点击顶部菜单 **工具** → **清单入库**
2. 确认 **Steam 路径** 已正确显示
3. **选择清单文件**：
   - 点击 **选择文件夹** 选择包含清单文件的文件夹
   - 或点击 **选择压缩包** 选择 7z/zip 格式的压缩包
4. 程序自动扫描并显示 Lua 文件、Manifest 文件、VDF 文件数量
5. （可选）配置 **访问令牌** 或 **成就数据 SteamID**
6. 点击 **开始 OpenSteamTool 入库**
7. 程序自动将 Lua 和 Manifest 文件复制到 Steam 对应目录
8. 若未开启热加载，可点击 **重启 Steam** 按钮重启 Steam 客户端

### 7.3 入库方式说明

| 选项 | 说明 |
|------|------|
| **热加载入库** | 开启后 Steam 运行时无需重启，OpenSteamTool 通过文件监视自动加载新 Lua |
| **高级模式** | 启用后会写入 Windows 注册表，通常用于 Denuvo/在线游戏 |
| **访问令牌** | 用于下载受 Steam 令牌保护的游戏或 DLC，仅在从 VDF 自动生成 Lua 时生效 |
| **成就数据 SteamID** | 用于拉取指定 Steam 账号的成就数据，仅在从 VDF 自动生成 Lua 时生效 |

### 7.4 游戏详情页入库

进入游戏详情页 → **入库 Steam** 标签页，也可使用 OpenSteamTool 内核入库。若游戏详情页已关联内置清单，程序会自动读取 `resources/manifest/{game_id}/` 下的文件。

## 八、假入库工具

假入库工具可在 Steam 库中模拟显示已拥有游戏，无需真实清单文件。适用于仅希望 Steam 库中显示游戏图标、而不需要实际下载游戏文件的场景。

### 8.1 前置要求

- 在【全局设置】中配置正确的 **Steam 安装路径**
- 已安装 **OpenSteamTool 内核 DLL** 到 Steam 根目录

### 8.2 使用步骤

1. 点击顶部菜单 **工具** → **假入库工具**
2. 在输入框中填写要假入库的游戏 **App ID**（多个 App ID 用逗号、空格或换行分隔）
3. （可选）填写游戏 **名称**，用于在列表中显示
4. 点击 **添加** 按钮将游戏加入假入库列表
5. 勾选需要假入库的游戏条目
6. 点击 **开始假入库**
7. 程序自动写入 OpenSteamTool 配置，使 Steam 库中显示对应游戏
8. 若 Steam 正在运行且已开启热加载，游戏会立即显示；否则重启 Steam 后生效

### 8.3 管理假入库列表

- **编辑**：修改已添加游戏的 App ID 或名称
- **删除**：从假入库列表中移除指定游戏
- **全选/取消全选**：批量操作列表中的游戏
- **导入/导出**：支持将假入库列表保存为 JSON 文件或从文件导入

### 8.4 注意事项

- 假入库仅改变 Steam 库显示，不会下载或生成实际游戏文件
- 部分在线游戏或受保护游戏可能无法通过假入库直接启动
- 建议与【清单入库】功能区分使用场景

## 九、全局设置

点击顶部菜单 **设置** → **全局设置**。

### 9.1 Steam 路径设置

- 配置 Steam 安装目录路径
- 用于清单入库与假入库功能

### 9.2 启动设置

- **启动后最小化到托盘**：程序启动后自动隐藏到系统托盘
- **关闭程序后隐藏在托盘**：点击关闭按钮后程序继续在后台运行（隐藏时自动释放图片缓存，恢复时重新加载）

### 9.3 OpenSteamTool 内核设置

- **内核状态**：显示 OpenSteamTool 内核 DLL 是否已安装到 Steam 目录
- **Steam 运行状态**：显示 Steam 客户端是否正在运行
- **安装/卸载内核**：手动安装或卸载 OpenSteamTool 内核 DLL
- **热加载入库**：开启后入库时不重启 Steam，依赖 OpenSteamTool 的文件监视自动加载新游戏
- **高级模式**：启用后会写入 Windows 注册表，通常用于 Denuvo/在线游戏
- **生成 opensteamtool.toml**：在 Steam 根目录生成 OpenSteamTool 默认配置文件，使用 wudrm 作为国内友好的 manifest 源
- **清理 SteamTools 残留**：清理旧版 SteamTools 可能残留的 DLL、stplug-in 目录和注册表项

### 9.4 语言设置

- 选择界面语言（简体中文 / English）
- 切换后立即生效，无需重启程序

### 9.5 恢复默认

一键恢复所有全局设置为默认值。

## 十二、常见问题与故障排除

### Q1: 程序无法启动？

- 确保系统为 Windows 10/11 64 位
- 检查是否安装了 WebView2 运行时
- 尝试以管理员身份运行
- 检查杀毒软件是否拦截

### Q2: 游戏下载进度卡在 0%？

- 检查网络连接是否正常
- 确认清单文件夹路径正确，包含有效的 `.manifest` 文件
- 检查磁盘空间是否充足
- 尝试更换下载路径到其他磁盘
- 检查 `%appdata%/SteamToolPlus/log/{game_id}/` 目录权限

### Q3: 补丁应用后游戏无法启动？

- 检查是否选择了正确的游戏文件夹
- 检查游戏是否放到纯英文路径
- 确保系统杀毒软件没有隔离删除补丁文件的 dll
- 尝试切换标准模式/高级模式
- 尝试切换稳定版/功能版
- 点击 **还原文件** 按钮恢复原始文件后重新配置

### Q4: 如何恢复原始游戏文件？

- 点击补丁页面的 **还原文件** 按钮
- 程序会自动删除补丁文件并恢复 `.bak` 备份

### Q5: 游戏库不显示游戏？

- 检查游戏数据文件 `%appdata%/SteamToolPlus/config/game.json` 是否存在
- 尝试重新导入游戏
- 检查游戏 exe 路径是否正确

### Q6: OpenSteamTool 入库后 Steam 库不显示游戏？

- 确认已开启 **热加载入库**，或点击 **重启 Steam** 按钮
- 检查 Steam 路径配置是否正确
- 确认 Lua 文件已正确写入 `Steam/config/lua/` 目录
- 检查 OpenSteamTool 内核 DLL 是否已安装到 Steam 根目录
- 查看 Steam 根目录 `opensteamtool/` 日志目录获取详细错误信息

### Q7: 什么是访问令牌？

- 访问令牌（Access Token）用于下载受 Steam 保护的某些游戏或 DLC
- 仅在没有现成 Lua 文件、从 VDF 自动生成 Lua 时生效
- 普通游戏留空即可

### Q8: 什么是成就数据 SteamID？

- 用于让 OpenSteamTool 拉取指定 Steam 账号的成就数据
- 仅在没有现成 Lua 文件、从 VDF 自动生成 Lua 时生效
- 留空时 OpenSteamTool 会默认使用 stats API 或内置预设 SteamID

### Q9: 背景图片无法显示？

- 确保图片格式为 JPG、PNG 或 WebP
- 检查图片文件是否损坏
- 尝试重新导入背景图片
- 删除 `%appdata%/SteamToolPlus/config/background.json` 后重启程序

### Q10: 程序打开后显示空白/透明窗口？

- 检查是否有杀毒软件拦截了程序
- 尝试删除 `%appdata%/SteamToolPlus/config/` 目录后重新启动程序

### Q11: 如何获取游戏的 Steam App ID？

- 访问 [SteamDB](https://steamdb.info/)
- 搜索游戏名称
- 查看页面上的 App ID
- 或在 Steam 商店页面 URL 中查看数字部分

### Q12: 局域网联机无法找到其他玩家？

- 确保所有玩家在同一局域网内
- 检查防火墙设置，允许程序通过
- 配置正确的广播地址（如 `192.168.1.0/24`）
- 确保所有玩家使用相同的补丁版本

### Q13: 为什么完整配置管理器中看不到 Overlay 配置？

- Overlay 属于功能版（实验版）DLL 的高级功能
- 请先在补丁页面基础配置中勾选“使用实验版（功能版）”，再打开完整配置管理器

### Q14: 配置保存后为什么其他窗口没有同步？

- 正常情况下配置保存后会自动广播同步事件
- 若未同步，请关闭并重新打开对应配置窗口
- 确保所有配置窗口都是通过同一游戏路径打开的

### Q15: 窗口隐藏到托盘后再次打开图片不显示？

- 程序已自动处理：隐藏时释放图片缓存，重新显示时自动重新加载
- 如仍有问题，尝试重启程序
